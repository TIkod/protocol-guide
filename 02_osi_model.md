# Модель OSI и стек TCP/IP (TICODE - https://ticode.online/)

## Зачем нужна модель OSI

Когда разные компании делали сети, возникал хаос — устройства одного вендора не работали с другими. В 1984 году ISO разработала модель **OSI (Open Systems Interconnection)** — стандарт, описывающий 7 уровней, на каждом из которых работают разные протоколы.

Модель OSI — это **концептуальная схема**, не реальная реализация. Реальные сети работают на **стеке TCP/IP** (4 уровня), но знание OSI помогает понять, что происходит с данными "в пути".

## 7 уровней OSI

```
┌─────────────────────────────────────────────────────────────┐
│  7. Прикладной (Application)    HTTP, FTP, SMTP, DNS        │
├─────────────────────────────────────────────────────────────┤
│  6. Представления (Presentation) TLS/SSL, JSON, XML, JPEG   │
├─────────────────────────────────────────────────────────────┤
│  5. Сеансовый (Session)         NetBIOS, RPC                │
├─────────────────────────────────────────────────────────────┤
│  4. Транспортный (Transport)    TCP, UDP                     │
├─────────────────────────────────────────────────────────────┤
│  3. Сетевой (Network)           IP, ICMP, OSPF              │
├─────────────────────────────────────────────────────────────┤
│  2. Канальный (Data Link)       Ethernet, Wi-Fi (802.11)    │
├─────────────────────────────────────────────────────────────┤
│  1. Физический (Physical)       Провода, оптика, радио      │
└─────────────────────────────────────────────────────────────┘
```

## Что делает каждый уровень

### Уровень 1 — Физический (Physical)

Передаёт **биты** по физической среде. Не знает ничего о содержимом.

- Кабели (витая пара, оптика, коаксиал)
- Беспроводные сигналы (Wi-Fi, Bluetooth)
- Преобразование: бит `1` → высокое напряжение, бит `0` → низкое

### Уровень 2 — Канальный (Data Link)

Передаёт **фреймы** между двумя соседними узлами в одной сети.

- MAC-адреса (физические адреса устройств)
- Обнаружение ошибок (CRC)
- Ethernet, Wi-Fi, Bluetooth

```
Фрейм Ethernet:
┌──────────┬──────────┬──────┬─────────────┬─────┐
│ MAC dst  │ MAC src  │ Type │   Payload   │ CRC │
│ 6 байт   │ 6 байт   │ 2 б  │ до 1500 б   │ 4 б │
└──────────┴──────────┴──────┴─────────────┴─────┘
```

### Уровень 3 — Сетевой (Network)

Маршрутизация **пакетов** между разными сетями. IP-адреса.

- IP (IPv4, IPv6) — логические адреса
- Маршрутизаторы работают на этом уровне
- Каждый пакет маршрутизируется независимо

```python
import socket

# IP адрес — адрес на уровне 3
ip = socket.gethostbyname("google.com")
print(ip)  # например: 142.250.185.46
```

### Уровень 4 — Транспортный (Transport)

Сквозная доставка **сегментов** от приложения к приложению. Порты.

- **TCP** — надёжно, с гарантией порядка
- **UDP** — быстро, без гарантий

```python
import socket

# Уровень 4: TCP сокет
tcp_sock = socket.socket(socket.AF_INET, socket.SOCK_STREAM)  # TCP

# Уровень 4: UDP сокет
udp_sock = socket.socket(socket.AF_INET, socket.SOCK_DGRAM)   # UDP
```

### Уровень 5 — Сеансовый (Session)

Управляет **сессиями** — установка, поддержание, завершение диалога.

В современном интернете эта роль часто встроена в другие уровни. TLS handshake — частично работа этого уровня.

### Уровень 6 — Представления (Presentation)

**Кодирование, шифрование, сжатие** данных.

- Преобразование форматов (Unicode → UTF-8 → байты)
- Шифрование TLS/SSL
- Сжатие gzip

### Уровень 7 — Прикладной (Application)

Протоколы для конкретных приложений — то, с чем работают разработчики.

- HTTP/HTTPS — веб
- SMTP — email
- DNS — имена
- FTP — файлы

## Как данные путешествуют: инкапсуляция

Каждый уровень **оборачивает** данные своим заголовком (инкапсуляция). На приёмнике — обратный процесс (декапсуляция).

```
Отправитель:
┌──────────────────────────────────────────────┐
│  Данные приложения: {"message": "hello"}     │  L7
├──────────────────────────────────────────────┤
│  HTTP заголовок + {"message": "hello"}       │  L7 + payload
├──────────────────────────────────────────────┤
│  TCP заголовок + HTTP + данные               │  L4
├──────────────────────────────────────────────┤
│  IP заголовок + TCP + HTTP + данные          │  L3
├──────────────────────────────────────────────┤
│  Ethernet заголовок + IP + ... + данные      │  L2
└──────────────────────────────────────────────┘
                     ↓ по проводу ↓

Получатель (снимает слои в обратном порядке):
L2: снимает Ethernet заголовок
L3: снимает IP заголовок
L4: снимает TCP заголовок
L7: читает HTTP и передаёт данные приложению
```

## Стек TCP/IP (реальная реализация)

TCP/IP — практическая реализация, используемая в интернете. Сжимает 7 уровней OSI до **4**.

```
OSI                        TCP/IP
─────────────────────────────────────────
7. Прикладной  ┐
6. Представлений├─────────► Прикладной (HTTP, DNS, FTP...)
5. Сеансовый   ┘
─────────────────────────────────────────
4. Транспортный ──────────► Транспортный (TCP, UDP)
─────────────────────────────────────────
3. Сетевой ───────────────► Интернет (IP, ICMP)
─────────────────────────────────────────
2. Канальный   ┐
1. Физический  ├──────────► Сетевого доступа (Ethernet, Wi-Fi)
               ┘
─────────────────────────────────────────
```

## Единицы данных на каждом уровне

| Уровень OSI | Единица данных | Пример |
|-------------|---------------|--------|
| 7, 6, 5 | Сообщение (Message) | HTTP запрос |
| 4 | Сегмент (TCP) / Датаграмма (UDP) | TCP сегмент |
| 3 | Пакет (Packet) | IP пакет |
| 2 | Фрейм (Frame) | Ethernet фрейм |
| 1 | Биты (Bits) | 10101010... |

## Практический пример: что происходит при GET запросе

```python
import httpx

# Это вызывает ВЕСЬ стек снизу вверх и обратно
response = httpx.get("https://httpbin.org/get")
```

Вот что происходит на самом деле:

```
1. [L7] Python формирует HTTP запрос:
   "GET /get HTTP/1.1\r\nHost: httpbin.org\r\n..."

2. [L6] TLS шифрует данные (HTTPS)

3. [L4] TCP разбивает на сегменты, добавляет порты
   src_port=54321, dst_port=443

4. [L3] IP добавляет адреса:
   src_ip=192.168.1.5, dst_ip=54.166.163.67

5. [L2] Ethernet добавляет MAC адреса:
   src_mac=aa:bb:cc:dd:ee:ff, dst_mac=<MAC роутера>

6. [L1] Биты уходят по кабелю/Wi-Fi

-- на стороне сервера всё в обратном порядке --

7. [L1] Биты получены
8. [L2] Извлечён IP пакет
9. [L3] Извлечён TCP сегмент  
10. [L4] Собраны все TCP сегменты
11. [L6] TLS расшифровывает
12. [L7] Nginx/uvicorn получает HTTP запрос
```

## Где работают сетевые устройства

| Устройство | Уровень OSI | Что делает |
|-----------|-------------|-----------|
| Кабель / Хаб | L1 | Передаёт биты, не думает |
| Коммутатор (Switch) | L2 | Пересылает фреймы по MAC |
| Маршрутизатор (Router) | L3 | Маршрутизирует пакеты по IP |
| Балансировщик (L4 LB) | L4 | Распределяет по TCP/UDP |
| Прокси / Nginx | L7 | Читает HTTP, принимает решения |

## Диагностика по уровням

```python
import subprocess
import socket

# L3 — проверить IP связность
subprocess.run(["ping", "-c", "3", "8.8.8.8"])

# L3/L4 — traceroute (маршрут пакетов)
subprocess.run(["traceroute", "google.com"])

# L4 — проверить порт
sock = socket.socket(socket.AF_INET, socket.SOCK_STREAM)
result = sock.connect_ex(("google.com", 443))
print("Порт 443 открыт" if result == 0 else "Закрыт")
sock.close()

# L7 — проверить HTTP
import httpx
r = httpx.get("https://httpbin.org/status/200")
print(f"HTTP статус: {r.status_code}")
```

## Запомни главное

- OSI — 7 уровней, **концептуальная модель** для понимания
- TCP/IP — 4 уровня, **реальный интернет**
- Данные оборачиваются заголовками при отправке (инкапсуляция)
- Данные разворачиваются на приёмнике (декапсуляция)
- Каждый уровень решает свою задачу и не знает о других
